## 第一步：注册 Supabase 并创建项目

### 1.1 注册 Supabase

1. 打开 [supabase.com](https://supabase.com)
2. 点击 **"Start your project"**
3. 选择 **"Continue with GitHub"** 登录（推荐，后面还要用 GitHub）

### 1.2 创建项目

1. 登录后点击 **"New project"**
2. 填写：
   - **Name**：`our-space`（随便起）
   - **Database Password**：设一个密码，**记下来！**（可以存浏览器密码管理器）
   - **Region**：选 `Southeast Asia (Singapore)`，离国内最近
3. 点击 **"Create new project"**
4. 等待 1-2 分钟，项目初始化完成

## 第二步：创建数据库表

1. 左侧菜单点击 **SQL Editor**（图标是 `>_`）
2. 点击 **"New query"**
3. 打开项目里的 `supabase.sql` 文件，**完整复制**内容粘贴进去
4. 点击右下角 **"Run"**
5. 看到 "Success. No rows returned" 就成功了 ✅

> 你也可以手动复制下面这段（和 supabase.sql 内容一样）：

<details>
<summary>点击展开 SQL（如果不想打开文件）</summary>

```sql
-- 纸条表
CREATE TABLE notes (
  id          BIGSERIAL PRIMARY KEY,
  author      TEXT NOT NULL CHECK (author IN ('he', 'she')),
  content     TEXT NOT NULL,
  parent_id   BIGINT REFERENCES notes(id) ON DELETE CASCADE,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 照片表
CREATE TABLE photos (
  id           BIGSERIAL PRIMARY KEY,
  storage_path TEXT NOT NULL,
  caption      TEXT DEFAULT '',
  uploaded_by  TEXT NOT NULL CHECK (uploaded_by IN ('he', 'she')),
  sort_order   INTEGER DEFAULT 0,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);

-- 倒计时表
CREATE TABLE countdowns (
  id          BIGSERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  target_date DATE NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_notes_created_at ON notes(created_at DESC);
CREATE INDEX idx_notes_parent_id ON notes(parent_id);
CREATE INDEX idx_photos_sort_order ON photos(sort_order, created_at DESC);
CREATE INDEX idx_countdowns_target_date ON countdowns(target_date);

-- RLS 策略
ALTER TABLE notes ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all on notes" ON notes FOR ALL USING (true) WITH CHECK (true);

ALTER TABLE photos ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all on photos" ON photos FOR ALL USING (true) WITH CHECK (true);

ALTER TABLE countdowns ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all on countdowns" ON countdowns FOR ALL USING (true) WITH CHECK (true);
```

## 第三步：创建照片存储桶并配置权限

### 3.1 创建存储桶

1. 左侧菜单点击 **Storage**
2. 点击 **"New bucket"**
3. 填写：
   - **Name of bucket**：`photos`（**必须**是这个名字，代码里写死了）
   - **Public bucket**：**打开开关** ✅
4. 点击 **"Create bucket"**

### 3.2 配置存储策略（重要！）

点击刚创建的 `photos` 存储桶 → 点击 **"Policies"** 标签 → 点击 **"New policy"** → 选择 **"For full customization"**

需要创建 **4 条策略**（Trae 原来的部署指南只写了 2 条，缺了关键的 DELETE 和 UPDATE）：

**策略 1：允许上传（INSERT）**
- Policy name：`Allow uploads`
- Allowed operation：✅ `INSERT`
- USING expression：`true`
- WITH CHECK expression：`true`
- → 点击 "Review" → "Save policy"

**策略 2：允许查看（SELECT）**
- Policy name：`Allow reads`
- Allowed operation：✅ `SELECT`
- USING expression：`true`
- → 点击 "Review" → "Save policy"

**策略 3：允许删除（DELETE）—— 这条是新增的！**
- Policy name：`Allow deletes`
- Allowed operation：✅ `DELETE`
- USING expression：`true`
- → 点击 "Review" → "Save policy"

**策略 4：允许更新（UPDATE）—— 这条是新增的！**
- Policy name：`Allow updates`
- Allowed operation：✅ `UPDATE`
- USING expression：`true`
- WITH CHECK expression：`true`
- → 点击 "Review" → "Save policy"

> ⚠️ **为什么需要 4 条策略？** 上传照片 = INSERT，查看照片 = SELECT，删除照片 = DELETE，更新照片 = UPDATE。只配 2 条的话，删照片和编辑照片会失败。

## 第四步：获取 API 密钥

1. 左侧菜单点击 **Project Settings**（齿轮图标 ⚙️）
2. 点击 **API**
3. 找到这两个值，**复制下来存好**：
   - **Project URL**：`https://xxxxxx.supabase.co`
   - **anon public** key：一长串字符（**注意是 `anon public`，不是 `service_role`！**）

## 第五步：创建 GitHub 仓库并推送代码

1. 打开 [github.com](https://github.com)，登录

打开终端，逐条执行：


## 第六步：配置 GitHub Secrets 和 Pages

### 6.1 配置 Secrets

代码推上去后，GitHub Actions 需要 Supabase 密钥才能构建：

1. 在仓库页面点击 **Settings**
2. 左侧菜单找到 **Secrets and variables** → **Actions**
3. 点击 **"New repository secret"**，添加两个密钥：

| Name | Value |
|------|-------|
| `VITE_SUPABASE_URL` | 第四步拿到的 Project URL（如 `https://xxxxxx.supabase.co`） |
| `VITE_SUPABASE_ANON_KEY` | 第四步拿到的 anon public key |

添加完后，你应该看到这两个密钥在列表里。

### 6.2 启用 GitHub Pages

1. 还是在 **Settings** 页面
2. 左侧菜单点击 **Pages**
3. **Source** 选 **"GitHub Actions"**（不是 "Deploy from a branch"！）

### 6.3 等待自动部署

1. 点击仓库顶部的 **Actions** 标签
2. 你会看到一个叫 "Deploy to GitHub Pages" 的 workflow 在跑（黄色 🔄）
3. 等它变成绿色 ✅ 就部署成功了
4. 如果变成红色 ❌，点进去看错误日志

> 第一次部署需要 1-2 分钟。以后每次 `git push` 都会自动重新部署。

## 常见问题

### Q: Supabase 免费版够用吗？

完全够用：
- 500MB 数据库（文字用几年）
- 1GB 存储（照片压缩后约 100-200KB/张，能存 5000-10000 张）
- 每月 2GB 带宽（两个人用绰绰有余）

> ⚠️ 连续 7 天不活跃会被暂停。记得每周至少打开一次。

### Q: 照片上传失败？

检查第三步的存储策略，确保 4 条都配了。最常见的错误是缺了 DELETE 策略。

### Q: 部署后页面空白？

检查 GitHub Secrets 的名字是否完全正确：`VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY`（注意大小写和前缀 `VITE_`）。

### Q: 部署后页面 404？

检查 Settings → Pages → Source 是否选的 "GitHub Actions"（不是 "Deploy from a branch"）。

### Q: 本地开发怎么启动？

```bash
cd /Users/lianrongjie/Desktop/开发/our-space
cp .env.example .env    # 编辑 .env 填入 Supabase 密钥
npm install
npm run dev             # 浏览器打开 http://localhost:5173
```

不填 Supabase 密钥也能跑，自动进入演示模式（数据存浏览器本地）。

### Q: 怎么更新代码？

```bash
cd /Users/lianrongjie/Desktop/开发/our-space
git add .
git commit -m "描述你改了什么"
git push
```

推上去后 GitHub Actions 自动重新部署，等 1-2 分钟生效。

### Q: 想换个好看的域名？

GitHub Pages 支持自定义域名。在 Settings → Pages → Custom domain 填入你买的域名，按提示配置 DNS 即可。不过先用默认地址也没问题。
